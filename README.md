# OpenWRT Router Compare (MT7981)

Интерактивная таблица сравнения бюджетных роутеров на базе **MediaTek MT7981**  
→ https://hattabbi4.github.io/openwrt-router-compare/

Таблица автоматически генерируется из CSV-файла и обновляется через Pull Request’ы.

---

## Как добавить новый роутер в таблицу

Добавление нового устройства **не требует знаний JavaScript или HTML**.  
Достаточно отредактировать один файл — `data.csv`.

---

## Кратко (если вы уже умеете пользоваться GitHub)

1. Сделайте **Fork** репозитория
2. Отредактируйте файл `data.csv` (добавьте одну строку)
3. Закоммитьте изменения
4. Создайте **Pull Request**

---

## Подробная инструкция

### 1. Сделайте Fork репозитория

1. Откройте репозиторий:  
   https://github.com/HattabbI4/openwrt-router-compare
2. Нажмите кнопку **Fork** (в правом верхнем углу)
3. Репозиторий появится в вашем аккаунте

---

### 2. Откройте файл `data.csv`

В вашем форке:

1. Перейдите в файл `data.csv`
2. Нажмите **Edit** (иконка карандаша)

---

### 3. Добавьте новую строку (роутер)

- **Одна строка = один роутер**
- **Порядок столбцов менять нельзя**
- Для подсветки можно использовать суффиксы `|good` и `|warn`

Пример строки:

```
MyRouter AX3000,ARM Cortex-A53,Wi-Fi 6,128 MB NAND,512 MB DDR4|good,4 внешние,Да (WAN)|good,USB 3.0|good,Нет,Нет,1 WAN + 3 LAN,Настольный
```

Правила:

- Не добавляйте лишние запятые
- Не используйте переносы строк внутри ячеек
- Если информации нет — указывайте `Нет` или `—`
- Если есть примечание — используйте `*` или `**`

---

### 4. Закоммитьте изменения

1. Внизу страницы введите сообщение коммита, например:

```
Add MyRouter AX3000
```

2. Нажмите **Commit changes**

---

### 5. Создайте Pull Request

1. В вашем форке нажмите **Contribute → Open pull request**
2. Проверьте параметры:
    - Base repository: `HattabbI4/openwrt-router-compare`
    - Base branch: `main`
3. Нажмите **Create pull request**
4. Кратко опишите, какой роутер вы добавили

---

## После этого

- Pull Request будет рассмотрен
- После принятия изменений:
    - данные попадут в ветку `main`
    - сайт автоматически обновится через GitHub Pages

---

## Формат данных (`data.csv`)

- **Первая строка** — названия характеристик
- **Первый столбец** — модель роутера
- Подсветка значений:
    - `|good` — преимущество (зелёный)
    - `|warn` — ограничение или нюанс (жёлтый)

---

## Причины, по которым Pull Request может быть отклонён

- Нарушен порядок столбцов
- Удалены или переименованы существующие поля
- В CSV добавлены HTML или Markdown теги
- Характеристики указаны без источников или пометок при сомнениях
